# Subagent presentation

Child Runs appear in transcript order as subagent rows with their delegated task available in expanded detail. Specialist rows use their user-facing role, such as Oracle, Librarian, codebase search, or review, while general children use Subagent and an available name.

Running, finished, failed, and cancelled states update the same row. Nested tool activity is shown beneath its owning subagent, and every underlying call appears once.

The agent identity remains primary while its lifecycle wording is muted, including in nested rows.
