# Composer editing and submission

The terminal composer supports cursor editing, multiline input, automatic height changes, submitted-input history, reverse history search, and opening a draft in an external editor. Users may drag its divider to set a manual height. Enter submits, while Shift+Enter, Ctrl+J, or a trailing backslash inserts a newline.

Empty input is not submitted. A failed submission keeps the draft so the user can correct it or try again.
