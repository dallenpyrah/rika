# Web research

Agents use `web_search` to obtain ranked current-web excerpts and `read_web_page` to turn a public HTTP or HTTPS page into bounded Markdown. Searches require an objective. `auto` chooses the best configured provider for a normal search; `compare` queries up to three suitable providers for disputed, recent, safety-critical, or high-impact public claims. Use `kind: "web"` for general research, `kind: "code"` for public semantic implementation examples, and `kind: "github"` for private or access-controlled repository research and exact GitHub-oriented code, repository, issue, pull-request, or commit research. Exa serves public semantic code examples through its Code Context route. GitHub is the sole GitHub-kind provider, so a GitHub-kind search is unavailable when GitHub is not configured. Access-controlled GitHub queries should be repository-qualified. Agents do not reformulate private or access-controlled terms as `code`, `web`, `compare`, or `read_web_page` after GitHub search is unavailable or insufficient; they use available local source, request access or source material, or report the limitation. For public material, an unavailable Exa Code Context or GitHub route may be replaced with another suitable kind, authoritative documentation, or public repository pages. Agents cite result URLs, identify source disagreement, and fetch authoritative public pages when snippets are insufficient rather than querying every provider every time.

Rika registers only configured providers. `web_search` is omitted when none are available, though an individual requested kind can still be unavailable when no configured provider supports it. `read_web_page` remains Parallel-backed and is omitted unless Parallel is configured.

Credentials in URLs and non-HTTP protocols are rejected. Missing service credentials, network or HTTP errors, invalid responses, extraction failures, and unavailable requested full content return typed failures; returned source text is bounded.
