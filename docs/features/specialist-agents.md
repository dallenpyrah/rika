# Specialist agents

The main agent can delegate durable Child Runs through Task, Oracle, Librarian, Review, and ReadThread tools. Task investigates the workspace and changes it only when assigned; independent investigations may run in parallel when genuinely independent. Oracle is a read-only advisor for high-level planning, architecture tradeoffs, difficult debugging analysis, and review of focused, already-gathered evidence, not primary codebase exploration. Librarian handles substantive external documentation, repository, and codebase research, including current API or library behavior, public implementation examples, compatibility and release evidence, and cited source verification. The main agent keeps a known URL, one-shot fact, or narrow external lookup direct; it delegates substantive external research to Librarian and keeps local workspace investigation with Task. Review assesses proposed or completed workspace changes for correctness, regressions, edge cases, and test coverage; ReadThread searches current or historical local Thread transcripts and returns a grounded free-form answer. The main agent uses ReadThread selectively when exact earlier context is missing rather than making transcript retrieval routine.

Task uses the selected mode's main route. Oracle, Librarian, Painter, Review, and ReadThread use its Oracle route. Root and delegation-enabled first-level agents may delegate, but second-level agents receive no delegation tools. Oracle and Review may call ReadThread but do not receive general delegation. Child Runs use pinned routes and capability snapshots; unavailable routes, missing snapshots, excessive nesting, cancellation, and failed children return explicit delegation results rather than widening access.
