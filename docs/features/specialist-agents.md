# Specialist agents

The main agent can delegate durable Child Runs through Task, Oracle, Librarian, Review, and ReadThread tools. Task can read, edit, and run processes; Oracle is a read-only advisor for planning, review, code comprehension, and debugging; Librarian reads network sources; Review inspects files without running processes; ReadThread searches current or historical local Thread transcripts and returns a grounded free-form answer. The main agent consults Oracle for difficult work and uses ReadThread selectively when exact earlier context is missing rather than making transcript retrieval routine.

Task uses the selected mode's main route. Oracle, Librarian, Painter, Review, and ReadThread use its Oracle route. Root and delegation-enabled first-level agents may delegate, but second-level agents receive no delegation tools. Oracle and Review may call ReadThread but do not receive general delegation. Child Runs use pinned routes and capability snapshots; unavailable routes, missing snapshots, excessive nesting, cancellation, and failed children return explicit delegation results rather than widening access.
