# Usage and cost

Relay durably owns execution, model call and attempt, usage, retry, compaction, child, wait, approval, and tool lifecycle facts. Rika does not copy those facts into an analytics schema or expose a separate analytics query surface; its local transcript and terminal totals are disposable product projections over Relay replay and inspection.

Rika derives model cost from Relay `model.usage.reported` usage and `model.attempt.completed` provider-cost observations, then exposes totals per Turn, Thread, and terminal session. A provider-reported USD amount is authoritative once for its model attempt. Otherwise Rika estimates cost from the bundled models.dev snapshot, using the reported provider, model snapshot, service tier, context tier, uncached input, cache reads, cache writes when known, and output. Rika does not maintain its own model prices or fetch live pricing while replaying durable events. Unknown models, service tiers, prices, currencies, and malformed token buckets remain unpriced instead of falling back to a generic rate. Child Run usage is attributed to its parent Turn and Thread by traversing Relay's durable child tree.

Token-derived values are estimates. Missing total input, uncached input, cache-read, cache-write, or output usage is unknown, not zero. Each source event is counted once by execution-scoped event identity, including on replay, recovery, and duplicate or out-of-order delivery. Thread totals cover all of that Thread's Turns. The terminal's global total covers Turns found in its bounded load of up to one hundred Threads. Rika withholds a total when that bounded replay is incomplete rather than combining overlapping or partially priced projections.

Thread deletion removes Rika's local product state but does not purge Relay's canonical execution records.
