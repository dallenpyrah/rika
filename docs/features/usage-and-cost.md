# Usage and cost

Rika derives model cost from Relay `model.usage.reported` events and exposes totals per Turn, Thread, and terminal session. An event-local `cost_usd` or `costUsd` field reported by the provider is authoritative. Otherwise Rika estimates cost from the bundled models.dev snapshot, using the reported provider, model snapshot, service tier, context tier, uncached input, cache reads, and output. Rika does not maintain its own model prices or fetch live pricing while replaying durable events. Unknown models, service tiers, prices, and malformed token buckets remain unpriced instead of falling back to a generic rate. Generic, nested, or cumulative cost fields are ignored. Child Run usage is attributed to its parent Turn and Thread by traversing Relay's durable child tree.

Token-derived values are estimates. Cache-write charges are excluded until Effect exposes OpenAI's cache-write usage; missing cache-write usage is not treated as a known zero. Each usage cursor is counted once — including on replay, recovery, and out-of-order or duplicate delivery — so cost never double-counts and survives event streams whose sequence numbers regress. Thread totals cover all of that Thread's Turns. The terminal's global total covers Turns found in its bounded load of up to one hundred Threads and falls back to the persisted per-Turn checkpoint sums when live recomputation is unavailable.

Persisted projections record the models.dev pricing version. When the catalog or calculator changes, completed Turns are rebuilt from Relay's durable events before their checkpoint becomes current. A complete rebuild may replace an old cost with either a higher or lower value. An interrupted or incomplete replay keeps the last complete persisted total.
