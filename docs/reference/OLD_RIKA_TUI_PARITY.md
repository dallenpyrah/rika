# Old Rika TUI parity audit

This is a source-level, user-visible parity audit of `rika-old` against the current `packages/tui`, `packages/app`, and `apps/rika`. It was captured on 2026-07-11. Paths under **Old evidence** are relative to `/Users/dallen.pyrah/projects/rika-old`; paths under **New evidence** are relative to this repository. Line references identify the implementation inspected, not an intended specification. “Exact” means the observable interaction is represented (not necessarily by identical architecture); “Partial” means a usable subset exists; “Missing” means no corresponding interaction/rendering was found.

The current visual baselines are synthetic renderer evidence, not proof of external parity: `packages/tui/test/fixtures/visual/` retains character frames and pairs them with cell-color PPMs and `.styles.json` OpenTUI span captures. The captures freeze foreground, background, and attributes at zero drift for Markdown, mode/selection colors, muted text, status colors, red/green diffs, tool states, queue, permission, thread/sidebar, and narrow overlays. They prove deterministic rendering of supplied models; only separately identified native PTY tests are live product evidence.

## Highest-priority parity gaps

1. **Missing:** old Markdown/OSC8 rendering and generic card/tool-group expansion.
2. **Partial:** `@@` interactive thread mentions, old image-extension coverage, rich Pierre diffs, grouped/status-rich tool cards, and the all-workspace thread switcher/sidebar preview.
3. **Partial:** transcript scroll/follow is modeled but not connected to rendering; queue steering works but lacks the old attached queue selection/cutout UX.
4. **Partial:** shortcuts omit effective old interactions, external-editor and reasoning-effort controls are absent, and composer geometry differs.
5. **Evidence gap:** synthetic cell-style fixtures now cover Markdown, multi-hunk/rename/binary/long diffs, tool states, queue, permission, thread/sidebar, and narrow overlays. They do not establish old/external parity, live OSC8/copy behavior, generic expansion, scroll/follow, errors, or teardown.

## Audit boundary

This audit intentionally compares user-visible behavior, not old architectural dependencies. It inspected tracked old TUI/controller/model/keymap/palette/diff code and the new TUI, app integration, CLI composition, tests, text frames, and PPM fixtures. A status is not upgraded merely because a reducer field or callback exists: rendering and an executable user path are required. Acceptance tests above are the closure criteria and should become native OpenTUI or visual tests where terminal behavior is involved.
